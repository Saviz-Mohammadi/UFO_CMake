CMake documentation

The UFO_CMake template handles most requirements for building a simple C++ project with CMake. However, when using CMake, there are a few key details we need to specify clearly, such as selecting the target toolchain and platform along with any special arguments for building the project. This section will guide you through these steps. This documentation includes sections below for each available toolchain, their recommended platforms, and the essential options and arguments. Please note that these represent only a subset of available concepts and options, and are the most important ones. For a comprehensive list, refer to the official CMake documentation or use the following command in your terminal:

cmake --help

Build

BUILDING THE PROJECT
The project includes a 'build' folder intended for generating project build artifacts. To initiate the build process from this directory, navigate to it using your terminal application. Then, execute the following commands, optionally appending any desired CMake arguments:

cmake "./../source"
cmake --build "./"

USING BUILD TOOLS
Remembering all the arguments and commands for building a CMake project can indeed be quite challenging. To alleviate this, we have developed platform-specific command-line helper tools located in the "build_tools" folder. Each platform includes a "config.json" file where you can conveniently configure your build arguments. These arguments will be simply copied over to the CMake executable. Once you have customized your build settings to your preference, you can initiate the generation and build process with a single command:

Windows (CMD)

./run_cmake.cmd

MacOS (Bash)

bash ./run_cmake.sh

Linux (Bash)

bash ./run_cmake.sh

RUNNING TESTS
If you decided to use this template for your CMake projects, then you're in luck because running tests is built-in using the Catch2 library. However, manually executing terminal commands to instruct CMake to run tests can be cumbersome. Therefore, we've developed a convenient utility that allows you to run tests directly from the terminal with ease. This tool is included in the "build_tools" folder, per each operating system. Simply navigate to the respective folder and execute the script as follows:

Windows (CMD)

./run_tests.cmd

MacOS (Bash)

bash ./run_tests.sh

Linux (Bash)

bash ./run_tests.sh

MSVC (Visual Studio)

DESCRIPTION:
The MSVC toolchain stands for "Microsoft Visual C++ Compiler", which contains a suite of tools including a compiler, linker, build system, resource manager, and more. These tools are primarily designed for and available exclusively on the Windows operating system. If you are using a Linux or MacOS machine, utilizing this toolchain can be either impossible or very challenging. This restriction is also why terms like "MSVC", "Visual Studio IDE", and "Windows" are often associated with each other, as they are closely interlinked.

INSTALLATION (Windows):
MSVC is currently exclusive to the Windows operating system. You have two installation options. You can either download the toolchain completely standalone by referring to the standalone installation link, or you can use the complete suite provided alongside the Visual Studio IDE installer. Whichever method you choose, make sure to select the category of "C++ for Desktop development" once the installer launches. This component will ensure that everything you need for developing C++ software is available to you.

MAKING TOOLCHAIN VISIBLE TO CMAKE:
After installing the toolchain on your machine, you need to ensure it is visible to CMake for generating your projects. The simplest method is to add the entire installation directory to your system's PATH. This allows CMake to automatically locate all necessary tools. This action is different from one platform and OS to another. You need to refer to the documentation section of your specific operating system to figure out how to add a certain tool to the system PATH.

Alternatively, you can add your installation folder's path to the list of paths that CMake will search to find your tool. To add multiple paths, separate them with a semi-colon (";") like this:

cmake -DCMAKE_PREFIX_PATH="/path1/to/toolchain/dir";"/path2/to/toolchain/dir"

Here is an example for where you can generally find the MSVC toolchain installation folder:

cmake -DCMAKE_PREFIX_PATH="C:\Program Files (x86)\Microsoft Visual Studio\2019\{Edition}\VC\Tools\MSVC\"

SPECIFYING GENERATOR (BUILD-SYSTEM):
A generator or build system is a scripting language shipped with each toolchain to streamline the process of compiling, linking, and generally building your project. It enbales developers to save their instructions in a build-script. It will then use this script to communicate with the compiler, linker, and other necessary tools on your behalf. MSVC utilizes MSBuild as its underlying build system. Visual Studio projects and solutions also take advantage of this build system behind the scenes. To instruct CMake to use this generator, specify the desired version of Visual Studio using the following command:

cmake -G "Generator" "path/to/top/level/CMakeLists.txt"

To obtain a comprehensive list of available names for your generator on your system, execute the "cmake --help" command. As you scroll down, you'll eventually come across a section that mentions "Generators". In that section you will find the names of all the generators that are currently supported on your platform. Keep in mind that just because they are supported, does not mean that they are installed. Here is an example of how to select MSVC (Visual Studio):

cmake -G "Visual Studio 17 2022" "path/to/top/level/CMakeLists.txt"

This action will also inform CMake that you intend to utilize the MSVC compiler, linker, and other associated tools from this toolchain, as they are highly compatible with this generator or build system. Some other generators or build-systems allow you to interact with other toolchain compilers, linkers, and tools. However, MSVC is very strict and does not allow this.

HELP WITH OPTIONS
For a concise and informative description of the customizable options and parameters specific to the MSVC (Visual Studio) toolchain, use the following command. Replace "Visual Studio 17 2022" with your installed version on your system:

cmake --help "Visual Studio 17 2022"

ARCHITECTURE SELECTION
One crucial consideration for Visual Studio is selecting the architecture, referred to as platform selection in MSVC (Visual Studio) terminology. By default, the target platform name matches that of the host system. You can specify the architecture using the "-A" argument. Below are the available platforms, each accompanied by a brief description:

Architecture Description
Win32 32-bit system
x64 64-bit system
ARM Arm 32-bit system
ARM64 Arm 64-bit system

For example, the following command generates an MSVC (Visual Studio) project configured for a 64-bit system architecture:

cmake -G "Visual Studio 17 2022" -A "x64"

CONFIGURATION TYPES
MSVC (Visual Studio) is a multi-configuration toolchain that enables you to specify and switch between various configuration types during your development. You can choose the configurations that your project starts with during the creation phase using the following command. The configurations are listed and separated by ';'. You can include or exclude any configuration types based on your requirements:

cmake -G "Visual Studio 17 2022" -DCMAKE_CONFIGURATION_TYPES="Debug;Release;MinSizeRel;RelWithDebInfo"
Configuration Description
Debug For debugging purposes
Release Optimized for best performance
MinSizeRel Optimized for minimal executable size
RelWithDebugInfo Release executable with debugging support

GCC

DESCRIPTION:
The GCC tool chain stands for "GNU Compiler Collection". GCC is a suite of compilers and tools developed by the GNU Project. It is widely used for compiling C, C++, and other programming languages mostly on Unix-based platforms including Linux, and MacOS. Despite the fact that GCC can be used on both Linux and MacOS platforms, but it is mostly adopted by the Linux community. As a result it is only natural to assume that when we talk about GCC, then the target platform in question is most likely Linux.

INSTALLATION (Windows):
For Windows platforms, you should refer to the MinGW toolchain, as GCC itself is not directly available for Windows.

INSTALLATION (MacOS):
For MacOS, the recommended method to install GCC is through the Homebrew package manager, where you can find the appropriate command on their website by searching for the keyword "gcc".

For detailed installation instructions, visit the official GNU Installation Page. Alternatively, you can download the binaries directly from the binaries page.

INSTALLATION (Linux):
For Linux users, many distributions include a package manager that simplifies the installation of the GCC toolchain. Below are commands for some popular package managers. Please note that some of these commands will install additional tools to ensure you have everything necessary for developing C/C++ projects under GCC.

For detailed installation instructions, visit the official GNU Installation Page. Alternatively, you can download the binaries directly from the binaries page.

apt (Debian/Ubuntu)

sudo apt install build-essential

MAKING TOOLCHAIN VISIBLE TO CMAKE:
After installing the toolchain on your machine, you need to ensure it is visible to CMake for generating your projects. The simplest method is to add the entire installation directory to your system's PATH. This allows CMake to automatically locate all necessary tools. This action is different from one platform and OS to another. You need to refer to the documentation section of your specific operating system to figure out how to add a certain tool to the system PATH.

Alternatively, you can add your installation folder's path to the list of paths that CMake will search to find your tool. To add multiple paths, separate them with a semi-colon (";") like this:

cmake -DCMAKE_PREFIX_PATH="/path1/to/toolchain/dir";"/path2/to/toolchain/dir"

Please note that on Unix platforms, files are often distributed across multiple locations based on their intended use and functionality. As a result, in most GCC installations, these locations are typically already added to the PATH or automatically detected by CMake.

SPECIFYING GENERATOR (BUILD-SYSTEM):
The GCC toolchain can be used and built with various build systems. The most widely used build system for GCC are Makefiles, which are shipped by default with the toolchain. To obtain a comprehensive list of available names for your generator on your system, execute the "cmake --help" command. As you scroll down, you'll eventually come across a section that mentions "Generators". In that section you will find the names of all the generators that are currently supported on your platform. Keep in mind that just because they are supported, does not mean that they are installed. Here is an example of how to select GCC with the build system as "Unix Makefiles":

cmake -G "Unix Makefiles" "path/to/top/level/CMakeLists.txt"

HELP WITH OPTIONS
For a concise and informative description of the customizable options and parameters specific to the GCC toolchain with "Unix Makefiles", use the following command:

cmake --help "Unix Makefiles"

ARCHITECTURE SELECTION
One crucial consideration for GCC is selecting the architecture. To specify the target architecture, one must pass the correct flag to the C++ compiler itself. As a result, in CMake you can specify the architecture to the compiler of the GCC toolchain by using the "-DCMAKE_CXX_FLAGS" argument. Below are some of the available architectures, each accompanied by a brief description:

Architecture Description
-m64 For 64-bit systems
-m32 For 32-bit systems

For example, the following command generates an GCC project configured for a 64-bit system architecture:

cmake -G "Unix Makefiles" "path/to/top/level/CMakeLists.txt" -DCMAKE_CXX_FLAGS="-m64"

CONFIGURATION TYPES
GCC operates as a single-configuration toolchain, requiring you to specify the desired build configuration upfront. You cannot switch between different configuration types during development. To define the configuration, use the "-DCMAKE_BUILD_TYPE" argument. If you need to develop with a different configuration type, you must create a separate build folder using CMake from the beginning. This allows you to include or exclude specific configuration types as needed for your project:

cmake -G "Unix Makefiles" "path/to/top/level/CMakeLists.txt" -DCMAKE_BUILD_TYPE="Debug"
Configuration Description
Debug For debugging purposes
Release Optimized for best performance
MinSizeRel Optimized for minimal executable size
RelWithDebugInfo Release executable with debugging support

MinGW

DESCRIPTION:
The MinGW tool chain stands for "Minimalist GNU for Windows". MinGW is a port of GCC for Windows. It includes necessary utilities and libraries to facilitate native Windows development using GCC. As a result of this, when developers use the word "MinGW", then we can safely assume that the target platform is Windows. MinGW is highly similar to GCC in almost every aspect. Therefore, concepts applicable to GCC also apply seamlessly to MinGW. MinGW-w64 is the newest project of MinGW that aims to bring more functionality and support to Windows.

INSTALLATION (Windows):
The latest iteration of the MinGW project, known as "MinGW-w64", incorporates the newest features from GCC. You can download the binaries directly from the official MinGW-w64 page. From personal experience, the best approach is to download the binaries from the WinLibs.com option offered by this page.

INSTALLATION (MacOS):
While MinGW is traditionally used for building Windows applications on Windows systems, it is possible to use this tool to create a cross-compilation environment. This enables us to compile Windows executables on non-Windows platforms such as MacOS. For MacOS, the recommended method to install MinGW is through the Homebrew package manager, where you can find the appropriate command on their website by searching for the keyword "mingw-w64".

For detailed installation instructions, visit the official GNU Installation Page. Alternatively, you can download the binaries directly from the binaries page.

INSTALLATION (Linux):

For detailed installation instructions, visit the official GNU Installation Page. Alternatively, you can download the binaries directly from the binaries page.

Clang/LLVM

DESCRIPTION:
The Clang toolchain project originated from Apple in 2007 as part of the LLVM project (Low Level Virtual Machine), aiming to replace GCC as the default compiler. Known for its modular design, Clang offers enhanced error messages and diagnostics compared to GCC. Although Clang is not limited to Apple platforms and supports Linux and Windows as well, it is often associated with Apple among developers.

INSTALLATION (Windows):
Note that some installations of MinGW or MSVC may include Clang/LLVM compilers and linkers, so you might already have them installed.

You can obtain the latest version of the Clang/LLVM project for various platforms by visiting their official website and navigating to the downloads section. On the downloads page you will find links to their GitHub releases.

INSTALLATION (MacOS):
For MacOS, the recommended method to install Clang/LLVM is through the Homebrew package manager, where you can find the appropriate command on their website by searching for the keyword "llvm". Note that some installations of GCC or MinGW may include Clang/LLVM compilers and linkers, so you might already have them installed.

You can obtain the latest version of the Clang/LLVM project for various platforms by visiting their official website and navigating to the downloads section. On the downloads page you will find links to their GitHub releases.

Note that some installations of GCC or MinGW may include Clang/LLVM compilers and linkers, so you might already have them installed.

You can obtain the latest version of the Clang/LLVM project for various platforms by visiting their official website and navigating to the downloads section. On the downloads page you will find links to their GitHub releases.

apt (Debian/Ubuntu)

sudo apt install llvm

SPECIFYING GENERATOR:
The Clang/LLVM toolchain is cross-platform, making it compatible with various build-systems. However, from my experience, it performs optimally when paired with the Ninja build-system generator. Simply specifying Ninja as the generator isn't sufficient to ensure Clang is automatically used, as other installed toolchains may be prioritized by CMake. Therefore, you must provide specific arguments to CMake for the compiler, linker, and other necessary settings. Below is the command you should use:

cmake -G "Ninja" "path/to/top/level/CMakeLists.txt" -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_LINKER=lld
